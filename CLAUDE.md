# ZeroGate — CLAUDE.md

Context for AI-assisted development on this project.

## What this project is

A self-hosted Zero Trust Network Access (ZTNA) platform. Core guarantees:
- **Zero inbound ports** on the EC2 security group — enforced by Terraform, never override.
- **MFA mandatory** for every user — never create a bypass, even for testing.
- **All secrets in AWS Secrets Manager** — never in code, docker-compose files, or committed `.env`.

## Project layout

```
infrastructure/   Terraform — AWS VPC, EC2, IAM, Secrets Manager
docker/           Docker Compose stack — all runtime services
  cloudflared/    Tunnel config (gitignored credentials.json + config.yml)
  authentik/      IdP — blueprints/ contains reusable flows
  guacamole/      Browser-based remote access — init/init.sh generates initdb.sql
  observability/  Grafana + Loki + Prometheus + Promtail + alert rules
scripts/          Operational bash scripts (all executable)
policies/         Access policy and security policy documentation
docs/             ARCHITECTURE, RUNBOOKS, ONBOARDING, TROUBLESHOOTING
```

## Critical invariants — never break these

1. `aws_security_group.main` in `infrastructure/ec2.tf` must have **zero** `ingress` blocks.
2. `docker-compose.yml` must **never** publish a port to `0.0.0.0` (only `127.0.0.1` in the override file).
3. All secrets come from `.env` (gitignored) populated by `bootstrap.sh` from Secrets Manager — never hardcode values.
4. Grafana provisioning YAMLs (`provisioning/alerting/`, `provisioning/datasources/`) do **not** expand shell `${}` variables — use literal values.
5. The `docker/guacamole/init/initdb.sql` is **gitignored** — generated by `./docker/guacamole/init/init.sh`.

## Common tasks

```bash
# Start everything
make up

# Check health
make health

# Security audit
make audit

# SSM shell to EC2 (no SSH)
make ssm

# Forward a port locally
make ssm-tunnel-9000   # Authentik
make ssm-tunnel-8080   # Guacamole
make ssm-tunnel-3000   # Grafana

# Rotate secrets
make rotate COMPONENT=grafana

# Full backup
make backup
```

## Environment variables

All documented in `.env.example`. Key ones:
- `AUTHENTIK_URL` — must match exactly what Cloudflare sends (no trailing slash)
- `GUACAMOLE_URL` — used as OIDC redirect URI in Authentik application config
- `GRAFANA_OIDC_CLIENT_SECRET` — set in Authentik, entered in `.env`

## When modifying Terraform

Always `terraform plan` before `apply`. Check that `IpPermissions: []` remains zero after any SG change:
```bash
make plan
# Review: confirm no ingress rules added to aws_security_group.main
```

## When modifying Docker Compose

- Keep all `healthcheck` blocks — cloudflared depends on them.
- Never remove `no-new-privileges: true` from security_opt.
- Keep `deploy.resources.limits.memory` on all services — t2.micro has 1 GB total.
- Any new service must join `zerogate-internal` network only.

## Loki log queries (useful for debugging)

```logql
# All auth failures
{service="authentik-server"} |= "login_failed"

# Guacamole connection events
{service="guacamole"} |= "GUAC_STATUS"

# All errors across stack
{project="zerogate"} |~ "ERROR|error|WARN" | json

# cloudflared tunnel health
{service="cloudflared"} |= "CONNECTED"
```
